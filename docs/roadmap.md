# Roadmap

Each phase below has a detailed spec in `docs/phases/`. Work through phases
in order; don't start a phase until the previous one's deliverables are done.

## [Phase 1 — Foundation](phases/phase-01-foundation.md)

Project setup, configuration, logging, Docker client, MCP server skeleton,
project structure, base models and exceptions.

## [Phase 2 — Docker Services](phases/phase-02-docker-services.md)

Containers, images, networks, volumes, system information.

## [Phase 3 — Remote Docker](phases/phase-03-remote-hosts.md)

Multiple hosts, SSH/TLS connections, host management, connection validation.

## [Phase 4 — Monitoring](phases/phase-04-monitoring.md)

Container stats, health checks, Docker events, resource monitoring.

## [Phase 5 — AI Diagnostics](phases/phase-05-ai-diagnostics.md)

Log collection, health analysis, AI context generation, recommendations.

## [Phase 6 — Automation](phases/phase-06-automation.md)

Scheduled tasks, cleanup, auto-healing, batch operations.

## [Phase 7 — Production Readiness](phases/phase-07-production.md)

Testing, documentation, performance tuning, security review, release.
